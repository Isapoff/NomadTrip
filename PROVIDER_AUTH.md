# Кабинет провайдера — вход по логину и паролю

Добавлен отдельный вход в кабинет провайдера (страница `/provider`), работающий
как админка: пока провайдер не авторизован, отображается экран входа; после входа —
дашборд с заявками, турами и аналитикой. Сверху показывается имя/компания и кнопка «Выйти».

## Что добавлено

**Бэкенд**
- Таблица `providers` в `db/schema.ts` (логин, хэш пароля, имя, компания, роль и т.д.).
- `api/lib/auth.ts` — хэширование паролей (scrypt) и подписанные сессионные токены (HMAC-SHA256).
- `api/context.ts` — читает сессионную cookie и подгружает текущего провайдера.
- `api/middleware.ts` — `protectedQuery` / `protectedMutation` для эндпоинтов «только для провайдеров».
- `api/routers/auth.ts` — процедуры `login`, `me`, `logout`, `profile`.

**Фронтенд**
- `src/hooks/useProviderAuth.ts` — состояние авторизации и действия (вход/выход).
- `src/components/provider/ProviderLogin.tsx` — форма входа.
- `src/pages/ProviderPage.tsx` — гейт: без входа показывает форму, после — кабинет.

Сессия хранится в httpOnly-cookie `nt_provider_session`, подписанной секретом `APP_SECRET`
(в dev есть запасной секрет, для продакшена обязательно задайте `APP_SECRET`).

## Установка

1. Создайте таблицу провайдеров в БД:

   ```bash
   npm run db:push        # применит изменения схемы через drizzle-kit
   ```

   (либо примените вручную `db/migrations/0001_providers.sql`)

2. Заведите учётные записи провайдеров:

   ```bash
   npm run db:seed:providers
   ```

   Можно одной командой пересоздать всю БД с провайдерами:

   ```bash
   npm run db:setup
   ```

## Учётные записи по умолчанию

| Логин   | Пароль      | Роль      |
|---------|-------------|-----------|
| `admin` | `nomad2025` | admin     |
| `demo`  | `demo1234`  | provider  |

> Обязательно смените пароли в проде — отредактируйте `db/seed-providers.ts`
> или добавьте свои записи в таблицу `providers` с хэшем из `hashPassword()`.

## Редактирование контента из кабинета (админка)

После входа в кабинете провайдера появились три вкладки-админки:

- **✎ Маршруты** — добавление/редактирование/удаление маршрутов. В форме есть
  все поля, важные для работы сайта: оценки 1–10 (Природа/Культура/Комфорт/
  Приключения/Стоимость) — они напрямую участвуют в алгоритме подбора WSM;
  координаты старта (для карты); поле «ID объектов» (через запятую, напр. `O1,O9`) —
  связывает маршрут с точками на карте.
- **✎ Жильё** — гостевые дома, юрты, отели (цена, рейтинг, координаты, контакты).
- **✎ Объекты** — точки интереса на карте (обязательны широта/долгота).

Изменения сразу попадают в базу и отражаются на публичных страницах
(«Маршруты», «Карта», «Жильё») и в подборе тура. Все операции записи защищены —
без входа провайдера API возвращает 401.

## Как добавить нового провайдера

Отредактируйте массив `SEED_PROVIDERS` в `db/seed-providers.ts` и снова запустите
`npm run db:seed:providers` — существующие записи обновятся, новые добавятся.
